# Figure 1 Caption — Nature Protocols Format

**Figure 1 | DAD end-to-end workflow for many-to-many protein–metabolite docking.**
The DAD (Dynamic Affinity Dock) pipeline processes multi-FASTA translations and metabolite SMILES inputs through 11 sequential stages organised into four colour-coded phases. *Input phase* (blue): FASTA ORF pool and SMILES library ingestion (Stage 1) followed by sequence quality control and dereplication by ≥95% identity clustering (Stage 2). *Triage phase* (vermillion): pre-docking biological triage (Stage 3) uses DeepTMHMM topology prediction to classify each ORF by transmembrane-helix count (nTM), signal-peptide status, and functional annotation, routing sequences to PASS, PASS_CLIPPED (periplasmic-domain extraction), or EXCLUDE (polytopic proteins, nTM ≥ 7). *Structure and ligand-preparation phase* (green): ColabFold structure prediction (Stage 4), P2Rank pocket detection (Stage 5), and RDKit-based ligand 3D embedding (Stage 6). *Docking and analysis phase* (pink): automatic docking-box configuration from P2Rank pocket coordinates (Stage 7), GNINA 1.3.2 docking with exhaustiveness = 32, num_modes = 9, seed = 0 (Stage 8), PLIP-based interaction profiling (Stage 9), and composite score aggregation into an N × M ranking matrix (Stage 10). *Output phase* (sky blue): automated per-pair HTML visualisation and project HTML report (Stage 11). No user-supplied hyperparameters are required. The decision diamond at Stage 3 indicates the DeepTMHMM-driven branching point that defines DAD's primary novelty relative to existing integrated-docking pipelines.
